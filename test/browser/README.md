# ชุดทดสอบชั้นเอกสาร (กดจริงบนหน้าเว็บ)

```
node test/browser/run.js          # ไฟล์ในเครื่อง — ใช้ก่อน commit
node test/browser/run.js --live   # เว็บจริงบน Pages — ใช้หลัง push
```

ไม่ต้องลงอะไรเพิ่ม — ใช้ Chrome ที่ติดมากับเครื่อง + WebSocket ใน Node (≥22)
ใช้เวลา ~40 วินาที (ช้ากว่า `test/run.js` มาก จึงแยกคำสั่ง ไม่รวมกัน)

| ชุด | ตรวจอะไร | เวลา |
|---|---|---|
| `test/run.js` | แกนคำนวณ (สูตร ตัวเลข วันที่) | ~1 วินาที |
| `test/browser/run.js` | ส.-งก.13 — กรอก กดปุ่ม ด่านบล็อก เอกสารออก | ~40 วินาที |
| `test/browser/check_janong.js` | หนังสือแสดงความจำนง — ด่านหนี้เดิมที่ตกจากเอกสาร | ~25 วินาที |
| `test/browser/check_negnote.js` | หมายเหตุแถวเงินต้นติดลบ (ทาง ง) โผล่/ไม่โผล่ | ~25 วินาที |
| `test/browser/check_tables.js` | ทุกตาราง — แถวบนหน้าจอ = แถวที่กระดาษพิมพ์ได้ + ตัวเลือกทุกตัวพิมพ์ออก | ~30 วินาที |
| `test/browser/check_failclosed.js` | ทำลายของจริงแล้วยืนยันว่าระบบ **ปฏิเสธที่จะพิมพ์** | ~30 วินาที |

ทุกตัวรับ `--live` เพื่อยิงเว็บจริงบน Pages แทนไฟล์ในเครื่อง

## ทำไมต้องมีชุดนี้

20-07-2569 ปิดบั๊กไป 13 ข้อ — **5 ข้อเป็นบั๊กที่เกิดระหว่างแก้บั๊กอื่น** และ 1 ข้อหลุดขึ้นเว็บจริง
บั๊กทั้งหมดเกิดที่ **ชั้นเอกสาร** ซึ่งตอนนั้นไม่มีเทสต์เลยสักข้อ
`node --check` ผ่าน · `test/run.js` เขียว · แต่เจ้าหน้าที่กดแล้วพิมพ์ไม่ได้

## กฎของ runner นี้

**เคสควบคุม (`expect: 'block'`) ต้องแดงครบ ไม่งั้น runner ปฏิเสธที่จะรายงานผลเขียว**

ตัวตรวจที่ fail ไม่ได้ = ตัวตรวจที่ไม่มีอยู่จริง — บทเรียนจากวันเดียวกันที่เขียนด่านแบบนั้นไป 2 ครั้ง
เคสควบคุม 3 ตัวปัจจุบัน: ไม่กรอกชื่อ · งวดเกินความจุตาราง · เกินเพดานบุคคลค้ำ

**เคส `expect: 'pass'` ต้องเห็นคำว่า "สำเร็จ" จริง** ไม่ใช่แค่ "ไม่มี ⚠️"
(เคยเขียนแบบหลัง แล้วได้เขียวปลอม 2 ข้อ ทั้งที่หน้าจอขึ้น `Failed to fetch` — เอกสารไม่ถูกสร้างเลย)

**เกณฑ์ตรวจต้องมาจากนอกสูตร** — เช่นเทียบยอดตารางกับ *วงเงินที่คนกรอก* ไม่ใช่เอาสูตรของโค้ดมาตรวจโค้ดตัวเอง

## เพิ่มเคสใหม่

แก้ `cases.js` อย่างเดียว — ประกาศ `expect`, `over` (สิ่งที่ต่างจาก `BASE`), และ `check()` ถ้าต้องตรวจตัวเลข

**ก่อนเชื่อเคสใหม่ ให้ทำลายโค้ดที่มันควรจับ แล้วดูว่ามันแดงจริง** เช่น:

```
# ย้อน num2 ใน sngk13_ui.js ให้ไม่ปัด → เคส 🎯 ต้องขึ้น
#   "ระบบผิดพลาด: ตารางใช้เงินกู้รวมได้ 99,999 บาท ไม่เท่าวงเงินที่พิมพ์ 100,000"
```

ถ้าทำลายโค้ดแล้วยังเขียว = เคสนั้นไม่ได้ทดสอบอะไร ลบทิ้งดีกว่าเก็บไว้ให้อุ่นใจผิด ๆ
